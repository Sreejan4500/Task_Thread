## 1. await returns the task result into your continuation

The video emphasizes two practical properties of **await**:

- It **yields** (no blocking), then resumes as a continuation when the awaited task completes.
- It **unwraps** the task result, so you can directly use the returned value in the subsequent steps.

This makes multi-step asynchronous flows readable without `ContinueWith(...)` chains or `Unwrap()`.

---

## 2. Example: sequential steps with multiple awaits

```csharp
using System.Text.Json;
using var client = new HttpClient();
var pokemonListJson = await client.GetStringAsync("https://pokeapi.co/api/v2/pokemon");

// Get the first Pokemon's url
var doc = JsonDocument.Parse(pokemonListJson);
JsonElement root = doc.RootElement;
JsonElement results = root.GetProperty("results");
JsonElement firstPokemon = results[0];
var url = firstPokemon.GetProperty("url").ToString();

// Get the first Pokemon's detailed info
var firstPokemonJson = await client.GetStringAsync(url);

// Get the weight and height
doc = JsonDocument.Parse(firstPokemonJson);
root = doc.RootElement;
Console.WriteLine($"Name: {root.GetProperty("name").ToString()}");
Console.WriteLine($"Weight: {root.GetProperty("weight").ToString()}");
Console.WriteLine($"Height: {root.GetProperty("height")}");
Console.WriteLine("This is the end of the program.");
Console.ReadLine();
```

## Summary table

| Concept | Role |
|--------|------|
| **await yields** | Registers continuation; does not block the caller. |
| **await unwraps Task&lt;T&gt;** | Makes the returned `T` available to the next steps. |
| **Continuation chain** | Becomes natural sequential code instead of nested continuations. |
