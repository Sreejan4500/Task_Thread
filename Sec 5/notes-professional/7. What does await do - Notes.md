# Notes: What does await do (Professional)

*From `7. What does await do.txt`.*

---

## 1. async/await is compiler transformation

The transcript explains that the compiler rewrites an `async` method into a **state machine**. The state machine:

- splits the method at each `await` into **states**
- stores locals as fields so they survive across awaits
- advances via a “move next” mechanism when awaited operations complete

---

## 2. “State” boundaries are at await points

Conceptually, the method becomes:

- State 0: from method start to first `await`
- State 1: from first `await` continuation to next `await`
- State N: remainder after the last `await`

---

## 3. Capturing synchronization context for continuations

The transcript emphasizes: at each await, the runtime captures the **SynchronizationContext** and later uses it to schedule the continuation on that captured context. This behavior is a key reason UI apps can often update UI after `await` without manual marshaling.

---

## Summary table

| Mechanism | Why it matters |
|----------|----------------|
| State machine | Preserves execution progress across async waits. |
| Locals lifted | Variables remain available after await. |
| Continuations | Code after await runs when the awaited task completes. |
| Context capture | Continuations can resume on UI context by default. |
