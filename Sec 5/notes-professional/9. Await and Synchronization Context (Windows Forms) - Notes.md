# Notes: Await and Synchronization Context (Windows Forms) (Professional)

*From `Await+and+Synchronization+Context(Windows+Forms).txt`.*

---

## 1. UI event handlers and async

The example uses `async void` event handlers (appropriate for UI events) that await an async `Task` method.

---

## 2. await and context capture

In Windows Forms, a `SynchronizationContext` exists for the UI thread. Awaiting inside a UI event handler captures this context, so the continuation after `await Task.Delay(...)` resumes on the UI thread, allowing direct UI updates without `Control.Invoke`.

---

## 3. Example code

```csharp
private async void button1_Click(object sender, EventArgs e)
{
    await ShowMessage("First Message", 3000);
}

private async void button2_Click(object sender, EventArgs e)
{
    await ShowMessage("Second Message", 4000);
}

private async Task ShowMessage(string message, int delay)
{
    await Task.Delay(delay);

    lblMessage.Text = message;
}
```
