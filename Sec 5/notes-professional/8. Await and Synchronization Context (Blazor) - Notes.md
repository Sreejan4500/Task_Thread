# Notes: Await and Synchronization Context (Blazor) (Professional)

*From `Await+and+Synchronization+Context(Blazor).txt`.*

---

## 1. Async event -> await -> UI state update

The example shows a Blazor click handler awaiting an async method, then updating component state (`currentTime`) and triggering a re-render via `StateHasChanged()`.

---

## 2. Example code (Razor component)

```razor
@page "/"

@currentTime

<button type="button" class="btn btn-primary" @onclick="OnDisplayTimeClicked">
    Display Time
</button>

@code {
    string currentTime = string.Empty;

    async void OnDisplayTimeClicked()
    {
        await DisplayTime();
    }

    async Task DisplayTime()
    {
        await Task.Delay(500);
        currentTime = DateTime.Now.ToString();
        StateHasChanged();
    }
}
```

---

## Summary table

| Concept | Role |
|--------|------|
| Event handler | Starts async flow from UI interaction. |
| `await Task.Delay` | Non-blocking delay. |
| State update | Assigns a new value to `currentTime`. |
| `StateHasChanged()` | Requests component re-render. |
