# Notes: Exception handling in PLINQ (Professional)

*From `4. Exception handling in PLINQ.txt`.*

---

## 1. Deferred execution affects where exceptions surface

PLINQ query operators execute during enumeration. Exceptions thrown inside the query (producer side) surface when the query is **consumed** (consumer side), not when the query is constructed.

Practically: wrap the enumeration / terminal operation in `try/catch`, not the query definition.

---

## 2. Aggregation model

PLINQ typically reports concurrent failures via `AggregateException`. You can handle individual inner exceptions using `AggregateException.Handle(...)`.

---

## 3. Example: catch on the consumer side

```csharp
var items = Enumerable.Range(1, 20);
ParallelQuery<int> evenNumbers = null;

evenNumbers = items.AsParallel().WithMergeOptions(ParallelMergeOptions.FullyBuffered).Where(x =>
{
    Console.WriteLine($"Processing number {x}; Thread Id: {Thread.CurrentThread.ManagedThreadId}");

    if (x == 5) throw new InvalidOperationException("This is intentional 5");
    if (x == 20) throw new ArgumentNullException("This is intentional 20");

    return (x % 2 == 0);
});

try
{
    evenNumbers.ForAll(item =>
    {
        Console.WriteLine($"{item}: Thread Id: {Thread.CurrentThread.ManagedThreadId}");
    });
}
catch (AggregateException ex)
{
    ex.Handle(e =>
    {
        Console.WriteLine(e.Message);
        return true;
    });
}
```
