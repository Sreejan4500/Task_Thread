# Notes: Basics of PLINQ (Professional)

*From `1. Basics of PLINQ.txt`.*

---

## 1. Opt-in parallelism for LINQ queries

PLINQ (Parallel LINQ) enables parallel execution over an `IEnumerable` by converting it to a `ParallelQuery` via `AsParallel()`. Under the hood, the source is partitioned and processed on multiple workers.

---

## 2. Ordering and determinism

Without additional ordering operators, results and processing order are not deterministic; interleaving depends on partitioning and scheduling.

---

## 3. Performance considerations

PLINQ adds overhead (partitioning/scheduling/merging). It is beneficial when per-element work is large enough (CPU-bound) or the data set is sufficiently large; otherwise sequential LINQ can be faster.

---

## 4. Example code (shape)

```csharp
var items = Enumerable.Range(1, 200);

var evenNumbers = items.AsParallel().Where(x =>
{
    Console.WriteLine($"Processing number {x}; Thread Id: {Thread.CurrentThread.ManagedThreadId}");
    return (x % 2 == 0);
});

Console.WriteLine($"Count: {evenNumbers.Count()}");
```

---

## Summary table

| Concept | Role |
|--------|------|
| `AsParallel()` | Enables PLINQ parallel execution. |
| Partitioning | Splits source into chunks for workers. |
| Merge | Combines partial results for consumption. |
| Overhead | Can dominate for cheap per-item work. |
