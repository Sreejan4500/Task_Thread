## 1. Producer/consumer model in PLINQ

PLINQ query execution resembles a producer/consumer pipeline:

- query operators (e.g. `Where`) run in parallel and **produce** results
- results are **merged** (buffered) for consumption by the downstream enumerator (e.g. `foreach`)

---

## 2. Controlling merge/buffer behavior

After `AsParallel()`, merge behavior can be tuned via `WithMergeOptions(ParallelMergeOptions ...)`:

- `AutoBuffered` (default): interleaves production/consumption
- `FullyBuffered`: completes production before consumption proceeds
- `NotBuffered`: minimizes buffering (still requires a handoff mechanism)

This applies to enumeration-based consumption (e.g. `foreach`).

---

## 3. Example code: FullyBuffered merge

```csharp
var items = Enumerable.Range(1, 200);

var evenNumbers = items.AsParallel()
    .WithMergeOptions(ParallelMergeOptions.FullyBuffered)
    .Where(x =>
    {
        Console.WriteLine($"Processing number {x}; Thread Id: {Thread.CurrentThread.ManagedThreadId}");
        return (x % 2 == 0);
    });

foreach (var item in evenNumbers)
{
    Console.WriteLine($"{item}: Thread Id: {Thread.CurrentThread.ManagedThreadId}");
}
```
