# Notes: Task Continuation — Continuation Chain and Unwrap (Professional)

*From `9. Task Continuation - Continuation Chain and Unwrap.txt`.*

---

## 1. Multi-step continuation chains

Real workflows often require several steps: e.g. (1) get list JSON, (2) get first item’s URL, (3) get details JSON from that URL, (4) parse and display. Each step can be a task that **continues from** the previous one via **ContinueWith**. So we get a chain: task1 → task2 → task3 → task4, with no blocking of the main thread.

---

## 2. Nested tasks: Task&lt;Task&lt;T&gt;&gt;

When a **continuation delegate** returns a **Task** (e.g. it calls **GetStringAsync**, which returns **Task&lt;string&gt;**), the **continuation’s** return type becomes **Task&lt;Task&lt;string&gt;&gt;** (or **Task&lt;Task&lt;T&gt;&gt;** in general). So we have a task that, when completed, produces **another** task. The next **ContinueWith** would then receive **t.Result** of type **Task&lt;string&gt;** (the inner task), not the string. To chain further and treat the inner task’s completion as the completion of the chain, we need to **flatten** the nesting.

---

## 3. Unwrap()

**Unwrap()** is an extension method on **Task&lt;Task&gt;** and **Task&lt;Task&lt;T&gt;&gt;**. It returns a **Task** (or **Task&lt;T&gt;**), which completes when the **inner** task completes and (for **Task&lt;T&gt;**) whose **Result** is the inner task’s result. So **Task&lt;Task&lt;string&gt;&gt;** becomes **Task&lt;string&gt;** after **Unwrap()**. We can then call **ContinueWith** on that and receive the string result (e.g. the details JSON) in the next continuation.

---

## 4. When to use Unwrap

Whenever a continuation returns a **Task** (e.g. an async method or **GetStringAsync**), the result of **ContinueWith** is **Task&lt;Task&lt;T&gt;&gt;**. Before attaching another **ContinueWith** that expects **t.Result** to be the final value (e.g. string), call **.Unwrap()** so the next continuation receives a **Task&lt;T&gt;** and **t.Result** is **T**. The video walks through the Pokémon example: list → first URL → details JSON (here **Unwrap** is used) → parse and output name, weight, height.

---

## 5. Key takeaway

Chaining continuations that return **Task**/ **Task&lt;T&gt;** produces **nested** tasks. Use **Unwrap()** to flatten so subsequent continuations operate on a single **Task&lt;T&gt;** and see the actual result type **T**. Without **Unwrap**, the next step would see a **Task** as **Result** instead of the underlying value.

---

## 6. Example code: nested ContinueWith + Unwrap

```csharp
using System.Text.Json;
using var client = new HttpClient();

var taskPokemonListJson = client.GetStringAsync("https://pokeapi.co/api/v2/pokemon");

var taskGetFirstPokemonUrl = taskPokemonListJson.ContinueWith(t =>
{
    var result = t.Result;
    var doc = JsonDocument.Parse(result);
    JsonElement root = doc.RootElement;
    JsonElement results = root.GetProperty("results");
    JsonElement firstPokemon = results[0];
    return firstPokemon.GetProperty("url").ToString();
});

var taskGetFirstPokemonDetailsJson = taskGetFirstPokemonUrl.ContinueWith(t =>
{
    var result = t.Result;
    return client.GetStringAsync(result);
}).Unwrap();

taskGetFirstPokemonDetailsJson.ContinueWith(t =>
{
    var result = t.Result;
    var doc = JsonDocument.Parse(result);
    JsonElement root = doc.RootElement;

    Console.WriteLine($"Name: {root.GetProperty("name").ToString()}");
    Console.WriteLine($"Weight: {root.GetProperty("weight").ToString()}");
    Console.WriteLine($"Height: {root.GetProperty("height")}");
});

Console.WriteLine("This is the end of the program.");
Console.ReadLine();
```

## Summary table

| Concept | Role |
|--------|------|
| **Continuation chain** | task1.ContinueWith(...).ContinueWith(...) for multi-step flow. |
| **Task&lt;Task&lt;T&gt;&gt;** | Produced when a continuation returns a Task&lt;T&gt;. |
| **Unwrap()** | Flattens Task&lt;Task&lt;T&gt;&gt; to Task&lt;T&gt;. |
| **When to Unwrap** | When the continuation returns a Task (e.g. async call) and you need to chain again with the inner result. |
