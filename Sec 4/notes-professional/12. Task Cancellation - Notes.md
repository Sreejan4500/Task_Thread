# Notes: Task Cancellation (Professional)

*From `12. Task Cancellation.txt`.*

---

## 1. Cancellation with Task: two approaches

With **Thread**, cancellation was done via a **shared variable** (e.g. **bool**) that the worker polled. With **Task**, that pattern still works (shared variable checked inside the task). The **recommended** approach is to use **CancellationTokenSource** and **CancellationToken**, which are the standard .NET mechanism and are supported by many APIs (e.g. **HttpClient.GetStringAsync**, **Task.Run**).

---

## 2. CancellationTokenSource and token

- **CancellationTokenSource** (e.g. **var cts = new CancellationTokenSource()**) is the object that **signals** cancellation (e.g. **cts.Cancel()**).
- **cts.Token** yields a **CancellationToken** that the task (and often the API) **observes**.
- **Task.Run(Action, CancellationToken)** (and overloads) accepts a token; the task can check **token.IsCancellationRequested** and exit (or throw). When the user or another component requests cancel, the caller invokes **cts.Cancel()**; the task sees the token and stops.

---

## 3. ThrowIfCancellationRequested

**token.ThrowIfCancellationRequested()** throws **OperationCanceledException** (or a subtype) if cancellation has been requested. So the task **faults** with a cancellation exception rather than completing normally. When the caller then uses **Wait()** or **Result**, that exception is thrown (wrapped in **AggregateException**). This is the standard way to “cooperatively” cancel and signal that cancellation occurred. If the task simply **returns** or **breaks** when cancellation is requested, it may **complete** (RanToCompletion) rather than **cancel** (Canceled); throwing makes the status and reason explicit.

---

## 4. CancelAfter (timeout)

**CancellationTokenSource.CancelAfter(int millisecondsDelay)** (and overloads) requests cancellation after a specified delay. So we can enforce a timeout: create a **CancellationTokenSource**, call **CancelAfter(timeout)**, pass **cts.Token** to **Task.Run** or to an async API (e.g. **GetStringAsync(url, token)**). If the work does not finish before the timeout, cancellation is requested and the task or API can stop. Useful for limiting how long an operation may run.

---

## 5. Why use CancellationToken

- **Consistency**: Many .NET and third-party async APIs accept **CancellationToken**. Passing the token allows those APIs to cancel in response to the same signal (e.g. user cancel or timeout).
- **Single mechanism**: One token can be passed through multiple layers; cancellation is coordinated without ad-hoc shared booleans.
- **Standard pattern**: Code that follows the token pattern is easier to integrate with the rest of the async ecosystem.

The lesson shows that **GetStringAsync** has an overload that takes a **CancellationToken**; when **CancelAfter** fires, the HTTP request is canceled and the task faults with a cancellation exception.

## 7. Example code: CancellationToken + cooperative cancellation

```csharp
using var cts = new CancellationTokenSource();
var token = cts.Token;

var task = Task.Run(Work, token);

Console.WriteLine("To cancel, press 'c'");
var input = Console.ReadLine();
if (input == "c")
{
    cts.Cancel();
}

task.Wait(); // propagates OperationCanceledException when ThrowIfCancellationRequested is used
Console.WriteLine($"Task status is: {task.Status}");

void Work()
{
    Console.WriteLine("Started doing the work.");

    for (int i = 0; i < 100000; i++)
    {
        if (token.IsCancellationRequested)
        {
            Console.WriteLine($"User requested cancellation at iteration: {i}");
            token.ThrowIfCancellationRequested();
        }

        Thread.SpinWait(30000000);
    }

    Console.WriteLine("Work is done.");
}
```

## Summary table

| Concept | Role |
|--------|------|
| **CancellationTokenSource** | Creates and signals cancellation; Cancel(), CancelAfter(ms). |
| **Token** | Pass to Task.Run and async APIs; task/API checks and obeys. |
| **IsCancellationRequested** | Poll in a loop; break or return to exit without throwing. |
| **ThrowIfCancellationRequested** | Throw so task is Canceled and Wait/Result propagate the exception. |
| **CancelAfter** | Timeout-based cancellation. |
| **Recommendation** | Use token for consistency with .NET async APIs. |
