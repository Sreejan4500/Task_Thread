# Notes: Exception handling in parallel loops (Professional)

*From `3. Exception handling in parallel loops.txt`.*

## 1. Exception aggregation model

In `Parallel.For`/`ForEach`/`Invoke`, iterations execute concurrently. When one or more iterations fault:

- the loop stops scheduling new iterations
- in-flight iterations typically complete/stop at a safe boundary
- exceptions are **aggregated** and rethrown to the caller as `AggregateException`

Therefore, exception handling is done by wrapping the parallel loop in `try/catch`.

---

## 2. Example: throw at i == 65

```csharp
int[] array = Enumerable.Range(0, 100).ToArray();

int sum = 0;
object lockSum = new object();

try
{
    Parallel.For(0, array.Length, (i, state) =>
    {
        lock (lockSum)
        {
            if (!state.IsExceptional)
            {
                if (i == 65)
                    throw new InvalidOperationException("This is on purpose.");

                sum += array[i];
                Console.WriteLine($"Current task id: {Task.CurrentId}; Is thread pool thread: {Thread.CurrentThread.IsThreadPoolThread}");
            }
        }
    });
}
catch (AggregateException ex)
{
    Console.WriteLine(ex);
}
```
