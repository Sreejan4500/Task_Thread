# Notes: Stop (Professional)

*From `4. Stop.txt`.*

---

## 1. Stop: cooperative early termination

Calling `ParallelLoopState.Stop()` requests that the loop **stop scheduling new iterations**. Iterations already in progress typically run to a safe stop/finish boundary. The video frames Stop as similar to the exception pathway, but **proactive** (developer-initiated) rather than reactive.

---

## 2. Detecting a Stop request

Other iterations can observe termination intent via `state.IsStopped` and skip work accordingly.

---

## 3. Example code: stop at i == 65

```csharp
int[] array = Enumerable.Range(0, 100).ToArray();

int sum = 0;
object lockSum = new object();

try
{
    Parallel.For(0, array.Length, (i, state) =>
    {
        lock (lockSum)
        {
            if (!state.IsStopped)
            {
                if (i == 65)
                    state.Stop();

                sum += array[i];
                Console.WriteLine($"Current task id: {Task.CurrentId}; Is thread pool thread: {Thread.CurrentThread.IsThreadPoolThread}");
            }
        }
    });
}
catch (AggregateException ex)
{
    Console.WriteLine(ex);
}
```
