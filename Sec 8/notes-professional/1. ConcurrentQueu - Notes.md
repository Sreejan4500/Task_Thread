# Notes: ConcurrentQueu (Professional)

*From `1. ConcurrentQueu.txt`.*

---

## 1. ConcurrentQueue<T>: a thread-safe FIFO queue

**Simple idea:**  
`ConcurrentQueue<T>` is a lock-free / internally synchronized queue intended for producer/consumer and general multi-threaded scenarios. Logically it is **FIFO** (first in, first out), like `Queue<T>`.

---

## 2. Dequeue => use TryDequeue to avoid “empty queue” races

**Simple idea:**  
With multiple threads, `Dequeue()` can race with other threads removing the last element. Instead, use:
- `TryDequeue(out T item)` which returns `false` if the queue is empty (no exception)

---

## 3. You typically don’t lock around the queue itself

**Simple idea:**  
`ConcurrentQueue<T>` provides the synchronization guarantees internally. You usually avoid writing your own `lock` around `Enqueue`/`TryDequeue`.

If your consumer does additional shared-state mutation beyond the queue operation, you may still need synchronization for *that* shared state.

---

## 4. Example code: processing a shared request queue

```csharp
using System.Collections.Concurrent;

ConcurrentQueue<string?> requestQueue = new ConcurrentQueue<string?>();

// 2. Start the requests queue monitoring thread
Thread monitoringThread = new Thread(MonitorQueue);
monitoringThread.Start();

// 1. Enqueue the requests
Console.WriteLine("Server is running. Type 'exit' to stop.");
while (true)
{
    string? input = Console.ReadLine();
    if (input?.ToLower() == "exit")
    {
        break;
    }

    requestQueue.Enqueue(input);
}

void MonitorQueue()
{
    while (true)
    {
        if (requestQueue.Count > 0)
        {
            if (requestQueue.TryDequeue(out var input))
            {
                Thread processingThread = new Thread(() => ProcessInput(input));
                processingThread.Start();
            }
        }
        Thread.Sleep(100);
    }
}

void ProcessInput(string? input)
{
    // Simulate processing time
    Thread.Sleep(2000);
    Console.WriteLine($"Processed input: {input}");
}
```

## Summary table

| Concept | Role |
|--------|------|
| `ConcurrentQueue<T>` | FIFO queue designed for concurrent producers/consumers |
| `Enqueue` | Thread-safe producer operation |
| `TryDequeue(out T)` | Safe consumer “remove” that returns `false` when empty |
| Internal synchronization | Usually no external `lock` needed for queue operations |
