# Notes: ConcurrentStack (Professional)

*From `2. ConcurrentStack.txt`.*

---

## 1. ConcurrentStack<T>: thread-safe LIFO

**Simple idea:**  
`ConcurrentStack<T>` is a concurrent, thread-safe stack with **LIFO** semantics (last in, first out).

It is intended for multi-threaded producer/consumer or general shared work distribution.

---

## 2. Push is the familiar operation

**Simple idea:**  
You still use `Push(T item)` to add an element.

---

## 3. Pop => TryPop to avoid empty-stack races

**Simple idea:**  
In concurrent scenarios, multiple threads may attempt to remove elements at the same time. The safe “remove” API is:
- `TryPop(out T item)` which returns `false` if the stack is empty

---

## 4. Example code: TryPop

```csharp
using System.Collections.Concurrent;

ConcurrentStack<int> stack = new ConcurrentStack<int>();

stack.Push(1);
stack.Push(2);
stack.Push(3);

if (stack.TryPop(out var value))
{
    Console.WriteLine($"Popped: {value} (expected: 3)");
}
```

## Summary table

| Concept | Role |
|--------|------|
| LIFO | Stack ordering semantics |
| `Push` | Thread-safe add |
| `TryPop(out T)` | Safe remove; returns false if empty |
| Internal synchronization | Collection operations are safe without external locks |
