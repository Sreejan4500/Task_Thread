# Notes: BlockingCollection (Professional)

*From `3. BlockingCollection and Producer & Consumer scenario.txt`.*

---

## 1. BlockingCollection<T>: producer/consumer buffer

**Simple idea:**  
`BlockingCollection<T>` is a thread-safe buffer that supports the **producer/consumer** pattern. It wraps another concurrent collection (e.g., `ConcurrentQueue<T>`), adding:
- **blocking semantics** (consumers wait when empty)
- optional **bounded capacity** (producers wait when full)

---

## 2. Bounding (upper capacity) and blocking (empty waits)

**Simple idea:**  
- **Bounded**: when `BoundedCapacity` is reached, `Add` blocks (or waits) until space becomes available
- **Blocking**: when consumers try to take and nothing is available, they block until items arrive

---

## 3. Producer uses `Add(...)`

**Simple idea:**  
Producers add items into the collection:
`collection.Add(item);`

---

## 4. Consumer uses `GetConsumingEnumerable()`

**Simple idea:**  
Consumers typically enumerate with:
`foreach (var item in collection.GetConsumingEnumerable())`

This enumerator:
- blocks when the collection is empty
- yields items as they become available
- completes when the producer signals completion

---

## 5. Example code: bounded + complete adding

```csharp
using System.Collections.Concurrent;

BlockingCollection<string?> collection = new BlockingCollection<string?>(requestQueue, 3);

// 2. Start the requests queue monitoring thread
Thread monitoringThread = new Thread(MonitorQueue);
monitoringThread.Start();

// 1. Enqueue the requests
Console.WriteLine("Server is running. Type 'exit' to stop.");
while (true)
{
    string? input = Console.ReadLine();
    if (input?.ToLower() == "exit")
    {
        collection.CompleteAdding();
        break;
    }

    collection.Add(input);

    Console.WriteLine($"Enqueued: {input}; queue size: {collection.Count}");
}

void MonitorQueue()
{
    
    foreach(var request in collection.GetConsumingEnumerable())
    {
        if (collection.IsCompleted) break;
        
        Thread processingThread = new Thread(() => ProcessInput(request));
        processingThread.Start();
        
        Thread.Sleep(2000);
    }
}

// 3. Processing the requests
void ProcessInput(string? input)
{
    // Simulate processing time
    Thread.Sleep(2000);
    Console.WriteLine($"Processed input: {input}");
}
```

## Summary table

| Concept | Role |
|--------|------|
| Producer/consumer wrapper | Adds coordination between adders and takers |
| Bounded capacity | Producer waits when buffer is full |
| Blocking behavior | Consumer waits when buffer is empty |
| `GetConsumingEnumerable()` | Consumer loop that blocks + takes automatically |
| `CompleteAdding()` | Signals “no more items”; ends the consumer loop |
